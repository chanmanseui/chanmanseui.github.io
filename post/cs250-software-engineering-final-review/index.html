PERT diagram && E-R Diagram

2022-01-01 3 min read

Software Process

PERT diagram [Project Scheduling]

Function

1. Model for project management and planning

a. Represent tasks and their dependencies.
b. Estimate amount of time for each task.
c. Compute the time for entire project and the critical path.

Example

1. Some basic concepts:

a. Oriented Graph – Each node is to represent a status; while each edge is
to represent a activity.
b. Critical path – path that takes longest.
c. Critical activity – Activity on the critical path.

2. TIME CALCULATION[ES, EF, LS, LF] --- Towarding to a specific activaty.

a. Eq.(A): Earliest Start (ES) Time = max{EFs of all immediate predecessors}
b. Eq.(B): Earliest Finish (EF) Time = ES + time of this activity
c. Eq.(C): Latest Start (LS) Time = LF – time of this activity
d. Eq.(D): Latest Finish (LF) Time = min{LSs of all immediate successors}
e. Initialization:
• ES of the first activity = 0.
• LF of the last activity = EF of the project.
f. Slack time[extra time for task]: Slack = LS – ES = LF – EF

3. A gereral method for analysing the PERT diagram[REALLY IMPORTANT]:

a. Initialize the ES of the first activity as 0, and then calculate the EF using Eq.(B).
b. Calculate the ES and EF of the successors from begin to end, according
to the Eq. (A) and Eq. (B).
c. Assign the EF of the project as the LF of the last activity, and the
calculate the LS using Eq.(C).
d. Calculate the LF and LS of the successors from begin to end, according
to the Eq. (D) and Eq. (C).
